Iterative Design and Evaluation

Overview

We set out to design a web interface for 1Flow, a platform that allows developers to receive in-platform feedback on mobile and web apps. After designing our hi-fi prototype, we received user feedback from fellow students, after which we iterated on our design to make it more user-friendly and improve user experience. We submitted our prototype to usertesting.com to receive additional feedback on our design, which we reflected on to consider what additional changes could be made for the next iteration.

We decided to implement a website because user testing feedback is not necessarily a service that would need to be accessed on-the-go. A website, which would more effectively display statistics, is a more intuitive platform for the company. We included what we considered to be the most essential functionalities in our mockup. Our prototypes include the ability to edit questionnaires that would be shown to the test users as well as a “User View” page so developers can see what their users see. We also included a “Stats” page that displays relevant statistics such as click-through rates in a graphical way along with users’ answers to the questionnaire.

Initial Sketches

Each of us made three sketches highlighting what we thought were some of the key pages and functionality for our individual visions of the site.

Lo-fi Prototypes

Taking all of our sketches into consideration, we consolidated the best parts of each of our designs and combined them into the Hi-fi prototype.

Everyone came up with the User View page and a result viewing page. We also included the “Sign in” page, a “My Apps” page, and a “Edit Questionnaire” page since they are important parts of our website.

Critique

These are the points of feedback that we addressed in our revised prototype:

  • 1Flow logo should take you back to My Apps and not the login page
    • How We Addressed It: 1Flow logo button now takes you back to My Apps
  • More intuitive back button for “Edit” and “Stats” pages than clicking My Apps again on top menu
    • How We Addressed It: We added a button reading “Back to My Apps” to both the “Edit” and “Stats” pages
  • Ability to click between views on stats page rather than scroll--something like "cards", or something else interactive
    • How We Addressed It: We changed the stats page to have cards that can be swiped through, and changed the questionnaire results to be dropdown answers so that the page is not a continuously scrolling page.
  • Add disclaimer that in "User View", the inputted data is not recorded as a statistic
    • How We Addressed It:We moved the disclaimer to be included in the header of each page in the User View.
  • Include progress measurement of some sort for user view (e.g. “1/10” or progress bar)
    • How We Addressed It:We added a progress measurement indicator to the user view (“1/3”)
  • Add some sort of indicator to show that you’re in user view (like a frame, make the view smaller than the screen size)
    • How We Addressed It:We added a large header that’s visible and indicates that you’re in user view.
  • Bring more emphasis to “Sign in to get started” on start page (make it a button)
    • How We Addressed It:We changed the text to be a large, blue button that’s easy to notice.

User Testing & Analysis

User 1 Video User 2 Video User 3 Video Written Feedback

The subtasks in our user tests were to sign into the website, interact with the questionnaire settings (edit a question, add a question, delete a question), and view the statistics page and “User View” pages. The most common error was with trying to find the “Questionnaire Edit” page—since the button was just labeled “Edit”, ⅔ users felt that was confusing and weren’t sure what exactly that button would have them editing. This is likely in part due to the inconsistency in labeling between the instructions and the prototype, but clarifying the express purpose of the “Edit” button can only help with clarity. Another user also recommended some kind of “instructions” page or popup that would remind a user how to use the site and what options are available to them. Based on this feedback, we could potentially add an instructions page and change the edit button to read “Edit Questionnaire.”
